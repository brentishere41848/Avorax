# Changelog

All notable Avorax changes should be documented here. Version entries avoid unsupported marketing claims and focus on implemented, testable behavior.

## Unreleased

### Changed

- Expanded dependency evidence reports with source-level lockfile package/integrity summaries and documented license-inventory metadata while keeping full release-host SBOM/license output explicitly required.
- Expanded release-binary ransomware-guard smoke coverage for `target\release\zentor_local_core.exe` from config-only proof to config/activity proof, adding strict `evaluate_ransomware_activity` IPC validation and bounded persisted-policy evaluation for caller-supplied activity observations.
- Added release-binary ransomware-guard configuration smoke coverage for `target\release\zentor_local_core.exe`, covering default listing, protected-root/trusted-process persistence, staged-write cleanup, broad-root rejection without overwrite, invalid persisted config fail-visible behavior, and unknown-field rejection.

- Added release-binary Quick Scan ZIP carrier review smoke coverage for `target\release\zentor_local_core.exe`, covering inert nested executable, deceptive nested executable, autorun bundle, autorun.inf command, and shortcut executable bundle ZIP review-only carriers.
- Added release-binary Quick Scan persistence/shortcut/disk-image carrier review smoke coverage for `target\release\zentor_local_core.exe`, covering inert registry autorun, URL/LNK shortcut, UNC shortcut, and ISO autorun review-only carriers.
- Added release-binary Quick Scan document/web carrier review smoke coverage for `target\release\zentor_local_core.exe`, covering inert autorun INF, EML attachment, Office query/macro, OOXML macro relationship, RTF/PDF active content, HTML/SVG, CHM/OneNote, and Office add-in review-only carriers.
- Added release-binary Quick Scan launch/installer carrier review smoke coverage for `target\release\zentor_local_core.exe`, covering ClickOnce, Java Web Start/JNLP, Windows scriptlet/SCT/WSC, and Windows Installer custom-action review-only carriers.
- Added release-binary Quick Scan AppInstaller carrier review smoke coverage for `target\release\zentor_local_core.exe`, with full-suite verifier/report-validator requirements.
- Added conservative static Windows App Installer/AppInstaller carrier review coverage for `.appinstaller` manifests that reference remote `.appx`, `.msix`, `.appxbundle`, or `.msixbundle` packages, with Quick Scan risky-file routing and review-only verdicts in confirmed-only mode.
- Shareable log/support-bundle credential redaction now also covers Basic/Digest/Negotiate/NTLM authorization values, Cookie/Set-Cookie headers, session/access/refresh token assignments, cookie/session diagnostic keys, and URL userinfo before export JSON is written.
- Normal local event-log export now redacts credential-like event details before writing `zentor-local-events.json`, using the same bounded shareable-export redaction family as support bundles while preserving raw in-app local event history for local audit.
- Small-threat MVP verification now has a first-class `Flutter shareable export credential-redaction tests` step, and passed non-skip-Flutter reports are rejected if shareable log/support-bundle credential-redaction evidence disappears.
- Support-bundle export now redacts credential-like diagnostics and event details before writing JSON, including sensitive keys, bearer tokens, password/token/API-key assignments, OpenAI-style keys, GitHub tokens, and JWT-shaped values.
- Small-threat MVP verification now has a first-class `Flutter support-bundle export tests` step, and passed reports that include Flutter coverage are rejected if support-bundle confirmation, busy-state, and privacy-flag evidence disappears.
- Added an explicitly confirmed diagnostic support-bundle export in Logs and Settings. The bundle is written atomically as local JSON, includes bounded app diagnostics and local event history, and records privacy flags that it contains no file contents, no quarantine payloads, no credentials, and no live malware while warning that event details can include local paths/errors.
- Small-threat MVP reports now include `generated_reports.no_eicar_harmless_threat`, and full-suite report validation parses that generated report to require the safe no-EICAR fixture policy, verified flow, and limitations instead of trusting step presence alone.
- Added a no-EICAR harmless-threat validation smoke that proves release local-core detect-only, confirmed-only quarantine, quarantine listing, and restore behavior without writing the standard EICAR string, weakening Defender, using live malware, or changing machine-wide settings.
- Flutter Protection now starts a bounded app-lifetime watch-poll scan loop when best-effort watcher roots are active, reports clean/limited/failure/threat outcomes through state and local events, stops the loop during protection shutdown/dispose/failure cleanup, and shows its status in the Protection tab without claiming service, kernel, or pre-execution coverage.
- Local Core now exposes a bounded `watch_poll_scan` IPC path for finite user-mode polling sessions with strict duration, interval, max-event, file-count, and depth limits, while continuing to report no persistent service monitoring and no kernel pre-execution blocking.
- Quick Scan risky file selection now includes additional script, persistence, shortcut, Office add-in/query, and OneNote carrier extensions, while ANE classifies `.ps1xml`, `.hta`, and `.wsf` into script-style analysis paths.
- Threat-intel import, signature/rule compilation, known-bad pack building, and pack validation now normalize supported category aliases and reject unsupported threat-category text before writing or accepting definition metadata.
- Pack validation now requires canonical threat-category spelling in already-built `.zsig` and `.zrule` files, matching the native engine schema while still letting compilers normalize aliases before output.
- Threat-intel action-policy input now normalizes aliases such as `review` and `observation_only` to native-supported pack values, and pack validation now rejects unsupported confidence, severity, signature type, action policy, file type, rule verdict/action, condition type, and PE import category metadata.
- Archive-entry heuristic aggregation no longer lets repeated weak archive metadata observations such as location or filename notes turn benign entry-count or total-content limit fixtures into threats by themselves.
- Local-core native verdict mapping now keeps `review_or_quarantine_by_policy` recommendations review-only unless stronger confirmed evidence maps the action to quarantine.
- Lightened the Flutter dark-theme secondary accent so text/status uses stay readable on Avorax dark surfaces.
- Refreshed Windows release-host prerequisite documentation after Flutter, Dart, Cargo, rustfmt, and Git became available locally; current blockers are Developer Mode/symlink support, Visual Studio Desktop C++ components, and a .NET SDK.

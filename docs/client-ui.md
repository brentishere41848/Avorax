# Avorax Client UI

Avorax is a native Flutter Material 3 app. The runtime UI is not a website,
browser, WebView, iframe, or embedded dashboard. This inventory is derived from
the Flutter sources under `apps/zentor_client/lib` and is used as the control
matrix for tabs, buttons, settings, confirmations, and visible engine state.

Flutter and Dart are available on the current Windows validation host. This
document marks source-level wiring separately from widget/runtime verification:
checkpoint 1717 verifies the high-risk Home, Scan, Protection, Quarantine,
Settings, Updates, Logs, and local-event control guards with focused Flutter
tests, and checkpoint 1869 verifies the full client Flutter suite both
sequentially and in parallel after the UI matrix runtime pass. Packaged desktop
click-through, OS picker dialogs, elevation dialogs, and Windows toast rendering
remain separate E2E work.

Checkpoint 2122 does not add a new UI control. Windows App Installer/AppInstaller
carrier review is surfaced through the existing Quick Scan and scan-result review
flows, with heuristic-only findings kept review-only in confirmed-only mode.

Checkpoint 2132 adds `tools/testing/validate-client-ui-inventory.py`, a
dependency-free source gate that validates the route matrix, desktop and mobile
navigation, Control Matrix row shape, documented verification status, and
source markers for high-risk tabs, buttons, settings, and custom engine status
controls before the small-threat MVP verifier can pass.

## First Screen

The app opens to Home (`/home`) after onboarding and does not open to an API
setup form. Home shows:

- Cloud status pill.
- Protection status pill.
- Run Quick Scan action.
- Run Full Scan action.
- Enable Protection or Stop Protection action.
- Update install action only when a verified update state says an update is
  available or being processed.
- Compact cards for Real-time protection, Native engine, AI advisory, Guard,
  Last scan, Quarantine, Updates, and recovery status.
- Recent activity from real local events only.

If Avorax Cloud is unavailable, the UI shows `Cloud Offline` without a red
blocking setup form. Cloud failure does not block local scanning or quarantine.

## Empty States

Production runtime must never show fake users, fake detections, fake charts, or
fake scan results. Empty states are explicit:

- No scan results.
- No threats found.
- No quarantined files.
- No allowlist entries.
- No activity yet.
- Update check failed.
- Unable to read platform info.

The Home headline, hero copy, real-time protection metric, Protection explanation, Settings Antivirus row, and Protection primary action label must not report or imply `Protected` unless engine evidence is also ready. If the malware engine is checking, unavailable, outdated, or errored, or if the native engine is not ready, the UI shows attention-needed/action-required status and explanation rather than a protected headline, ready-assets copy, enabled metric, or local-only reassurance even when protection settings are enabled. The Protection explanation, Settings Antivirus row, and controller `ProtectionStatus.protected` transition use the same combined malware/native engine evidence gate before ready-assets or protected-state copy. Partial protection must remain visibly limited or labeled as driver self-test required instead of disabled or fully enabled.

Device protection must not depend on any unrelated product mode.
